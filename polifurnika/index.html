📌 Bab 1

Apa itu JASP?

Pengantar, keunggulan, perbandingan, instalasi, dan implementasi JASP untuk penelitian pengajar.

Tentang JASP

JASP (Jeffreys's Amazing Statistics Program) adalah perangkat lunak statistik yang gratis, open source, dan dikembangkan oleh Universitas Amsterdam. Didesain untuk ilmuwan sosial — cukup drag variabel, output muncul real-time dalam format APA. Bayesian dan Frequentist berdampingan dalam satu antarmuka.

✅ Keunggulan JASP
Gratis selamanya, antarmuka visual tanpa kode, output APA otomatis, Bayesian terintegrasi, R sudah bundled di dalamnya.
⚠️ Keterbatasan
Tidak mendukung .xlsx langsung (perlu konversi ke .csv), fitur lanjutan membutuhkan pemahaman R untuk kustomisasi lebih jauh.

Perbandingan dengan Software Lain

Perlu Kode✗ TidakYaSebagianYa

💡
Untuk pengajar dan tendik, JASP adalah pilihan terbaik: gratis, antarmuka visual, output langsung bisa disalin ke laporan Word atau PowerPoint tanpa format ulang.

Langkah Instalasi JASP

  • 1

    Buka jasp-stats.org/download/

    Pilih installer sesuai sistem operasi: Windows (.exe), macOS (.dmg), atau Linux (.AppImage).

  • 2

    Jalankan Installer

    Double-click file installer. Ikuti wizard instalasi. JASP tidak memerlukan R yang terpisah — sudah terintegrasi dalam paket instalasi.

  • 3

    Buka JASP & Cek Versi

    Buka JASP, pergi ke Menu (☰) → About JASP. Pastikan menggunakan versi terbaru (0.19+).

  • 4

    Atur Bahasa Indonesia

    ☰ → Preferences → Interface → Language → Bahasa Indonesia. Restart JASP agar perubahan berlaku.

JASP untuk Penelitian Pengajar

Sebagai pengajar yang aktif meneliti, JASP menawarkan keunggulan nyata dibanding software lain — bukan sekadar "lebih mudah dipakai", tetapi secara spesifik dirancang untuk alur kerja riset akademik.

📄

Output APA Siap Manuskrip

Tabel dan grafik langsung keluar dalam format APA 7th — siap disalin ke jurnal, skripsi, atau laporan PTK tanpa format ulang manual.

⚖️

Bayesian + Frequentist Sekaligus

Dua pendekatan statistik berdampingan dalam satu analisis. Sangat relevan untuk penelitian modern yang mensyaratkan pelaporan Bayes Factor di samping p-value.

🔁

Reproducibility via File .jasp

Simpan file .jasp — semua analisis, variabel, dan output tersimpan. Kirim ke reviewer atau rekan: mereka bisa membuka dan memverifikasi ulang hasil Anda persis sama.

🌐

Koneksi OSF (Open Science)

JASP terintegrasi langsung dengan Open Science Framework (OSF) — platform standar internasional untuk pre-registrasi dan berbagi data penelitian terbuka.

📊

Fokus Visualisasi Data untuk Penelitian

JASP menghasilkan grafik berkualitas publikasi langsung dari menu klik — tidak perlu kode R atau Excel tambahan. Jenis grafik yang paling berguna untuk pengajar-peneliti:

Histogram distribusi data Boxplot perbandingan kelompok Raincloud plot (distribusi + outlier) Scatter plot + garis regresi Heatmap korelasi antar variabel Means plot ANOVA (interaksi faktor) QQ plot uji normalitas residual

💡 Semua grafik dapat diekspor resolusi tinggi (300+ DPI) langsung dari JASP via ☰ → Preferences → Results → DPI — siap masuk jurnal tanpa re-render.

Implementasi: Alur Penelitian dengan JASP

Berikut alur kerja nyata menggunakan JASP dari data mentah hingga laporan siap publikasi — cocok untuk penelitian PTK, survei, maupun eksperimen di lingkungan pendidikan.

  • 1

    Siapkan data di Excel → Simpan sebagai .csv

    Susun data dengan baris = responden/sampel, kolom = variabel. Simpan via Save As → CSV UTF-8. Pastikan tidak ada baris judul ganda atau sel kosong di tengah data.

  • 2

    Buka di JASP → Verifikasi tipe variabel

    ☰ → Open → pilih file .csv. Cek header kolom — pastikan variabel nominal/ordinal/scale sudah tepat. Klik ikon tipe variabel di header untuk koreksi jika perlu.

  • 3

    Eksplorasi data dengan grafik awal (Descriptives)

    Descriptives → Descriptive Statistics → drag semua variabel utama → centang Distribution plots dan Boxplots. Lihat pola distribusi, outlier, dan nilai ekstrem sebelum uji utama.

  • 4

    Pilih & jalankan uji statistik sesuai tujuan penelitian

    Gunakan pohon keputusan di Bab 4 untuk memilih uji yang tepat (t-test, ANOVA, regresi, korelasi). Setiap output sudah mencantumkan effect size dan interpretasi siap kutip.

  • 5

    Buat grafik bermakna untuk laporan

    Tambahkan grafik yang mendukung narasi penelitian: boxplot untuk perbandingan kelompok, scatter + garis regresi untuk hubungan antar variabel, means plot untuk hasil ANOVA. Atur DPI 300+ di Preferences terlebih dahulu.

  • 6

    Ekspor ke Word → Susun laporan/artikel

    ☰ → Export Results → Word — semua tabel APA dan grafik terkompilasi dalam satu file .docx. Tinggal tambahkan narasi interpretasi di antara tabel dan grafik yang sudah terformat rapi.

  • 7

    Simpan file .jasp untuk reprodusibilitas

    ☰ → Save → simpan file .jasp. File ini menyimpan data + seluruh analisis + output. Lampirkan ke submission jurnal atau bagikan ke rekan sebagai bukti transparansi penelitian.

🎯
Kesimpulan untuk Pengajar-Peneliti: JASP bukan sekadar alternatif SPSS yang gratis. Dengan output APA otomatis, dukungan Bayesian, file .jasp yang reproducible, dan grafik siap publikasi — JASP adalah alat yang memang dirancang untuk ekosistem penelitian akademik modern.

Memahami Antarmuka JASP

Kenali setiap elemen antarmuka JASP dan cara mengkustomisasinya.

Elemen Utama Antarmuka

☰ Hamburger Menu (kiri atas)

Buka/simpan file, ekspor hasil ke Word/PDF, koneksi OSF (Open Science Framework), dan Preferences.

⊞ Top Bar Analisis

Akses semua menu analisis: Descriptives, T-Tests, ANOVA, Regression, Frequencies, Factor, dsb.

✚ Plus Button (kanan atas)

Tambah modul analisis lanjutan: SEM, Meta-Analysis, Machine Learning, Network Analysis, dan lainnya.

⚙ Preferences

Kustomisasi tema, bahasa, desimal, resolusi grafik, dan nilai hilang. Akses via ☰ → Preferences.

📋
Tata letak layar: Panel kiri = Control Panel (pilih variabel & opsi analisis). Panel kanan = Output Panel (hasil analisis muncul otomatis, real-time saat opsi diubah).

Kustomisasi Preferences

📊

Data Preferences

Atur editor default, ambang batas kategori vs skala, dan kode missing value kustom (mis. 99, 999).

📋

Result Preferences

Tampilkan p-value tepat, jumlah desimal, resolusi grafik, dan latar transparan/putih untuk grafik.

🖥️

Interface Preferences

Pilih tema light/dark, bahasa (Indonesia tersedia), ukuran zoom, dan font untuk aksesibilitas.

✅
Pro Tip: Atur resolusi grafik ke 300+ DPI di Preferences → Results sebelum mulai analisis agar grafik siap publikasi tanpa perlu redo.

Konsep Dasar Statistik

Fondasi yang wajib dipahami sebelum memilih dan menjalankan uji statistik di JASP.

1 · Skala Pengukuran Data

Jenis data menentukan uji statistik yang tepat. Gunakan tabel berikut sebagai panduan cepat.

Tabel 1. Skala Pengukuran dan Contoh Variabel Penelitian
SkalaSimbol JASPCiri KhasContoh VariabelOperasi Valid
Nominal🟠 NKategori tanpa urutanJenis kelamin, agama, jenis kayuMode, frekuensi, Chi-square
Ordinal🔵 OKategori dengan urutan, jarak tidak samaTingkat kepuasan (1–5), peringkat kelasMedian, Mann-Whitney, Spearman
Interval🟢 SJarak sama, tidak ada nol mutlakSuhu °C, skor IQ, nilai ujianMean, SD, t-test, ANOVA
Rasio🟢 SJarak sama + nol mutlakBerat badan, tinggi, penghasilanSemua operasi statistik
💡
Di JASP, interval dan rasio digabung menjadi tipe Scale (S). Jika ragu antara ordinal vs scale, tanyakan: "Apakah jarak antar nilai benar-benar sama?" — jika ya, pilih Scale.

2 · Ukuran Pemusatan & Penyebaran

μ

Mean (Rata-rata)

Jumlah semua nilai dibagi banyak data. Sensitif terhadap outlier. Gunakan jika data berdistribusi normal dan skala interval/rasio.

Md

Median

Nilai tengah setelah data diurutkan. Lebih robust terhadap outlier. Pilihan utama untuk data ordinal atau yang tidak normal.

Mo

Modus

Nilai yang paling sering muncul. Satu-satunya ukuran pemusatan yang valid untuk data nominal (kategori).

σ

Standar Deviasi

Rata-rata jarak setiap nilai dari mean. Semakin besar SD, semakin bervariasi data. Selalu dilaporkan bersama mean.

Sk

Skewness

Ukuran kemiringan distribusi. Nilai ideal: −1 hingga +1. Skewness > 2 menunjukkan distribusi sangat tidak normal.

Ku

Kurtosis

Ukuran "keruncingan" distribusi. Nilai ideal: −2 hingga +2. Penting untuk asumsi normalitas sebelum uji parametrik.

3 · Distribusi Normal & Asumsi Uji Statistik

✓

Data Berdistribusi Normal

Histogram berbentuk lonceng simetris. Skewness dan Kurtosis mendekati 0. Uji Shapiro-Wilk: p > .05.

→ Gunakan uji parametrik

t-test, ANOVA, Pearson, Regresi

!

Data Tidak Normal

Histogram menceng (skewed). Skewness atau Kurtosis ekstrem. Uji Shapiro-Wilk: p < .05.

→ Gunakan uji non-parametrik

Mann-Whitney, Wilcoxon, Kruskal-Wallis, Spearman

4 · Hipotesis Nol (H₀) dan Hipotesis Alternatif (H₁)

H₀ (Hipotesis Nol): Pernyataan yang menyatakan tidak ada perbedaan atau hubungan. Contoh: "Tidak ada perbedaan rata-rata diameter kayu antar jenis kayu."

H₁ (Hipotesis Alternatif): Pernyataan yang ingin dibuktikan peneliti. Contoh: "Ada perbedaan rata-rata diameter kayu antar jenis kayu."

Jika p < .05 → Tolak H₀

Ada bukti statistik yang cukup untuk mendukung H₁. Perbedaan/hubungan dianggap signifikan secara statistik.

Jika p ≥ .05 → Gagal Tolak H₀

Tidak cukup bukti untuk menolak H₀. Bukan berarti H₀ benar — hanya berarti data tidak cukup membuktikan H₁.

5 · Effect Size — Ukuran Praktis Temuan

📏
p-value ≠ Effect Size. Nilai p hanya memberi tahu apakah efek ada, bukan seberapa besar efeknya. JASP secara otomatis menampilkan effect size di setiap output.

📈 Bab 6

Grafik, Tabel & Ekspor

Cara membuat grafik publikasi, memilih visualisasi yang tepat untuk penelitian, kustomisasi, dan ekspor ke Word/PDF.

Jenis Grafik di JASP

📊

Histogram & Bar

Via Descriptives → Plots → Distribution plots. Centang Display density untuk kurva normal overlay.

📦

Boxplot & Raincloud

Descriptives → Plots → Boxplots. Aktifkan Raincloud plot untuk visualisasi distribusi yang lebih kaya.

🔵

Scatter Plot

Regression → Correlation → Plots. Atau via Descriptives → Plots → Scatter plots untuk matrix scatter.

🌡️

Correlation Heatmap

Regression → Correlation → centang Heatmap. Visualisasi matriks korelasi antar semua variabel.

📉

Means Plots (ANOVA)

ANOVA → Plots → Marginal Means. Untuk visualisasi interaksi antar faktor dengan garis dan error bars.

🔬

Residual & QQ Plot

Regression → Plots → centang Residuals vs Fitted dan Q-Q Plot untuk cek asumsi normalitas residual.

💡
Tips Ekspor: Klik kanan grafik → Copy image atau Save image as. Untuk resolusi tinggi, atur ke 300+ DPI di ☰ → Preferences → Results sebelum analisis dimulai.

Walkthrough: Grafik Perbandingan Kelompok (Raincloud)

Raincloud plot adalah grafik terbaik untuk membandingkan kelompok dalam penelitian — menggabungkan distribusi, boxplot, dan titik data sekaligus dalam satu visualisasi.

  • 1

    Buka Descriptives

    Klik Descriptives di top bar → pilih Descriptive Statistics.

  • 2

    Drag variabel ke kotak yang tepat

    Drag variabel kontinu (mis. Efisiensi_persen) ke Variables. Drag variabel kelompok (mis. Jenis_Kayu) ke Split by.

  • 3

    Aktifkan Raincloud Plot

    Di panel kiri, scroll ke bagian Plots → centang Boxplots → muncul opsi Raincloud → aktifkan. Grafik langsung muncul real-time di panel kanan.

  • 4

    Kustomisasi tampilan

    Centang Mean pada opsi boxplot untuk menambahkan tanda rata-rata. Atur warna via tema di Preferences jika perlu konsistensi visual laporan.

  • 5

    Ekspor grafik

    Klik kanan grafik di panel kanan → Save image as → pilih PNG/SVG. Atau gunakan ☰ → Export Results → Word untuk ekspor semua grafik sekaligus.

    Buka Regression → Correlation

    Klik Regression di top bar → pilih Correlation.

  • 2

    Drag variabel

    Drag semua variabel kontinu yang ingin dilihat korelasinya ke kotak Variables (mis. Diameter, Kadar_Air, Ketebalan, Efisiensi).

  • 3

    Aktifkan Heatmap dan Scatter Matrix

    Di panel kiri centang: Heatmap (visualisasi korelasi berwarna) dan Scatter plots (matrix scatter antar semua variabel). Langsung terlihat pasangan variabel mana yang paling kuat hubungannya.

  • 4

🧪 Bab 7

Walkthrough Uji T & ANOVA

Panduan langkah demi langkah menjalankan Uji T Independen, Uji T Berpasangan, dan ANOVA Satu Jalur di JASP menggunakan data latihan Furnitur.

A · Independent Samples T-Test (Uji T Dua Kelompok Independen)

🎯
Tujuan: Membandingkan rata-rata dua kelompok yang berbeda (tidak berpasangan). Contoh: apakah rata-rata Kadar_Air_persen berbeda antara kayu Jati dan Meranti?
T

Persyaratan

Variabel dependen: Scale (interval/rasio) Variabel grup: Nominal (2 kategori) Idealnya: data mendekati normal per grup
  • 1

    Buka menu T-Tests

    Klik T-Tests di top bar JASP → pilih Independent Samples T-Test.

  • 2

    Atur variabel

    Drag Kadar_Air_persen ke kotak Dependent Variables. Drag Jenis_Kayu ke kotak Grouping Variable. JASP akan otomatis mendeteksi dua kategori.

  • 3

    Centang opsi penting

    Di panel kiri, centang: Descriptives (statistik per kelompok), Welch's t-test (lebih robust jika variansi tidak sama), dan Effect size (Cohen's d).

  • 4

    Cek Assumption Checks

    Buka bagian Assumption Checks → centang Normality (Shapiro-Wilk) dan Equality of Variances (Levene's). Jika Shapiro-Wilk p < .05, pertimbangkan Mann-Whitney U.

  • 5

    Baca hasil output

    Perhatikan kolom: t (statistik uji), df (derajat bebas), p (nilai signifikansi), Cohen's d (besar efek). Jika p < .05 dan Cohen's d ≥ 0.5, temuan bermakna secara statistik dan praktis.

✍️
Template laporan APA: Hasil uji t independen menunjukkan perbedaan yang signifikan antara kelompok Jati (M = X.XX, SD = X.XX) dan Meranti (M = X.XX, SD = X.XX), t(df) = X.XX, p = .XXX, Cohen's d = X.XX.

B · Paired Samples T-Test (Uji T Berpasangan / Pre-Post)

🎯
Tujuan: Membandingkan rata-rata dua pengukuran pada subjek yang sama. Contoh: nilai pre-test vs post-test pelatihan, atau pengukuran sebelum dan sesudah intervensi.
  • 1

    Buka Paired Samples T-Test

    Klik T-Tests di top bar → pilih Paired Samples T-Test.

  • 2

    Pasangkan variabel

    Drag pasangan variabel sekaligus (misal: Nilai_Pre + Nilai_Post) ke kotak Paired Variables. Setiap baris adalah satu pasang pengukuran.

  • 3

    Centang opsi

    Centang Descriptives, Effect size (Cohen's d), dan buka Assumption Checks → centang Normality of differences (Shapiro-Wilk).

  • 4

    Baca hasil

    Fokus pada: nilai t, p, dan Cohen's d. Jika data tidak normal (Shapiro-Wilk p < .05), gunakan Wilcoxon Signed-Rank sebagai alternatif non-parametrik.

C · One-Way ANOVA (Komparasi 3+ Kelompok)

🎯
Tujuan: Membandingkan rata-rata tiga kelompok atau lebih sekaligus tanpa inflasi error tipe I. Contoh: apakah rata-rata Efisiensi_persen berbeda antara Jati, Meranti, dan Pinus?
  • 1

    Buka ANOVA

    Klik ANOVA di top bar → pilih ANOVA (bukan Repeated Measures). Untuk data non-normal, pilih Kruskal-Wallis di menu Non-Parametrics.

  • 2

    Atur variabel

    Drag Efisiensi_persen ke Dependent Variable. Drag Jenis_Kayu ke Fixed Factors.

  • 3

    Centang Effect Size & Post Hoc

    Di panel kiri: centang η² (eta-squared). Buka Post Hoc Tests → geser Jenis_Kayu ke kanan → centang Tukey (untuk kelompok sama) atau Games-Howell (jika variansi tidak homogen).

  • 4

    Cek asumsi

    Buka Assumption Checks → centang Homogeneity of variances (Levene's) dan Q-Q plot. Jika Levene's p < .05, laporkan Welch's ANOVA sebagai alternatif.

  • 5

    Baca tabel ANOVA & Post Hoc

    Tabel ANOVA: perhatikan F, df, p, dan η². Tabel Post Hoc Tukey: cari pasangan kelompok mana yang berbeda signifikan (pTukey < .05).

  • 6

    Buat grafik Descriptives

    Buka Descriptives Plots → geser Jenis_Kayu ke Horizontal Axis. Centang Error bars: 95% CI untuk menampilkan interval kepercayaan per kelompok.

✍️
Template laporan APA: Hasil One-Way ANOVA menunjukkan perbedaan yang signifikan pada Efisiensi_persen antara kelompok jenis kayu, F(df1, df2) = X.XX, p = .XXX, η² = .XX. Analisis post hoc Tukey mengindikasikan bahwa [kelompok A] secara signifikan lebih tinggi dibandingkan [kelompok B] (pTukey = .XXX).

💻 Bab 8

R for JASP

Akses kode R di balik setiap klik JASP — untuk reprodusibilitas, fleksibilitas, dan belajar R bertahap.

JASP sebenarnya berjalan di atas R — setiap klik di JASP menghasilkan kode R di belakang layar. Fitur R for JASP memungkinkan Anda melihat, menyalin, dan memodifikasi kode R tersebut untuk kontrol penuh atas analisis.

ℹ️
Anda tidak perlu instalasi R atau RStudio terpisah. R sudah terintegrasi dalam JASP. Fitur ini tersedia via Hamburger → R for JASP atau ikon </> di setiap output.

Mengapa Belajar R Sintaks di JASP?

🔁

Reprodusibilitas

Simpan skrip R untuk analisis yang dapat diulang kapan saja dengan dataset berbeda — lebih andal dari klik manual.

🎨

Fleksibilitas

Modifikasi grafik dan tabel beyond menu — warna kustom, anotasi, ukuran font, tema ggplot2.

📚

Belajar R Bertahap

JASP sebagai jembatan — lihat kode R otomatis, pelajari sintaksnya, kembangkan sendiri tanpa tekanan.

Cara Mengakses Kode R di JASP

  • 1

    Jalankan Analisis Apapun

    Misalnya: Descriptives → pilih variabel. Output muncul di panel kanan.

  • 2

    Klik Ikon </> di Output

    Di setiap tabel atau grafik, ada ikon kode di pojok kanan atas. Klik untuk melihat kode R-nya secara langsung.

  • 3

    Buka R Console di JASP

    ☰ → R for JASP. Muncul terminal R interaktif langsung dalam JASP — bisa mengetik perintah R.

  • 4

    Jalankan & Modifikasi

    Salin kode dari output, tempel ke R console, ubah parameter, dan jalankan untuk hasil kustom.

Ekspor Kode R dari JASP

Sejak JASP pertama dikembangkan, fokus utamanya adalah merancang antarmuka grafis (GUI) yang mempertahankan keunggulan sintaks — seperti reprodusibilitas — tanpa mengharuskan pengguna menulis atau menghasilkan kode. Namun, seiring waktu tim JASP menyadari bahwa banyak pengguna ingin mengakses sintaks R di balik setiap analisis. Fitur ini tersedia mulai JASP versi 0.17 ke atas.

ℹ️
Sintaks R di JASP saat ini bekerja di dalam JASP itu sendiri — dapat digunakan untuk mengontrol GUI dan berjalan di konsol R JASP. Ke depan, sintaks R yang dihasilkan JASP juga akan dapat dieksekusi langsung di RStudio. Sumber: JASP FAQ

R Syntax Mode: Tombol Pertama (JASP 0.17)

Dengan rilis JASP 0.17, sebuah tombol baru ditambahkan ke menu analisis inti. Selain mengganti nama, menduplikasi, menghapus analisis, dan melihat file bantuan, pengguna JASP kini dapat mengakses function call R yang digunakan untuk menghitung hasil. Ini adalah langkah penting pertama menuju integrasi yang lebih lengkap antara JASP dan R.

🎯 Tujuan Utama

Salah satu tujuan utama JASP adalah membuat analisis statistik lebih mudah diakses dengan menghilangkan kebutuhan pemrograman komputer, sehingga mahasiswa dapat fokus pada interpretasi, bukan implementasi kode.

🔗 Sinkronisasi Dua Arah

Kode R dan GUI JASP saling terhubung secara real-time: mengubah opsi di GUI otomatis memperbarui kode R, dan mengubah kode R lalu menekan Ctrl+Enter (Cmd+Enter di Mac) otomatis memperbarui GUI.

Cara Menggunakan R Syntax Mode

  • 1

    Jalankan Analisis di JASP

    Pilih analisis apa pun dari menu atas, misalnya Descriptives atau T-Tests, lalu atur variabel dan opsi sesuai kebutuhan. Output muncul real-time di panel kanan.

  • 2

    Klik Tombol R di Panel Analisis

    Di setiap analisis yang sedang aktif, klik tombol R yang muncul di menu analisis tersebut. Kode R yang setara dengan pilihan GUI Anda akan langsung ditampilkan.

  • 3

    Salin & Bagikan Kode

    Salin kode R dan bagikan kepada rekan atau mahasiswa — mereka dapat melihat persis opsi apa yang digunakan dalam analisis, atau mem-paste ke JASP mereka untuk mereproduksi hasil yang sama.

  • 4

    Edit Kode untuk Kontrol Lanjutan

    Ubah parameter di dalam kode R lalu tekan Ctrl+Enter — GUI JASP akan otomatis menyesuaikan dan menghasilkan output baru. Ini memungkinkan eksplorasi analisis yang sangat fleksibel.

Contoh Penggunaan: Deskriptif & ANOVA

📊

Contoh 1 — Paired T-Test dengan Descriptives

Misalnya pada dataset Moon & Aggression dari JASP Data Library, saat opsi tabel deskriptif dan raincloud plot dicentang pada analisis Paired T-Test, tombol R akan menampilkan function call lengkap beserta semua pengaturannya. Setiap perubahan pada GUI langsung tercermin di kode R secara otomatis.

T-Tests → Paired Samples T-Test Klik tombol R → lihat function call
🧪

Contoh 2 — Three-Way Mixed ANOVA

Untuk analisis kompleks seperti Three-Way Mixed ANOVA (dataset Bugs), sintaks R sangat membantu pengajar: alih-alih menjelaskan berbagai submenu dan antarmuka repeated measures cells, cukup bagikan sintaks R kepada mahasiswa. Mereka membuka dataset, membuka R console module di JASP, paste sintaks, klik Add analysis — dan analisis langsung terbuka dengan semua opsi sudah terisi otomatis.

ANOVA → Repeated Measures ANOVA Paste sintaks → Add analysis Semua opsi terisi otomatis

4 Keunggulan R Syntax Mode di JASP

📤 Berbagi Analisis

Pengguna dapat menyalin kode R yang dihasilkan JASP dan membagikannya kepada rekan — penerima langsung mengetahui opsi apa saja yang digunakan tanpa perlu membuka file .jasp.

🔁 Reprodusibilitas

Tempel kode R ke dalam JASP Anda sendiri untuk mereproduksi analisis secara persis — memastikan konsistensi hasil penelitian dan kemudahan verifikasi.

⚙️ Kontrol via Kode

Edit kode R untuk mengontrol GUI, atau ubah GUI untuk menghasilkan kode R yang berbeda. Kode selalu tersinkron dengan GUI sehingga perubahan di satu sisi langsung berdampak ke sisi lain.

🎓 Alat Pengajaran

Pengajar dapat dengan mudah berbagi pengaturan analisis kepada mahasiswa dan mendemonstrasikan konsep pemrograman R dalam lingkungan yang ramah pengguna tanpa setup tambahan.
